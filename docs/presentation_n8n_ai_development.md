### 오픈소스 워크플로우 자동화 플랫폼

| 특징 | 설명 |
|------|------|
| **노코드/로우코드** | 코딩 없이 시각적으로 워크플로우 설계 |

| **셀프호스팅** | 사내 서버에 설치하여 데이터 보안 확보 |

### 비용 비교
```
┌─────────────────────────────────────────────────────┐
│  Zapier (SaaS)     : 월 $50~$800 (사용량 기반)      │
│  Make.com          : 월 $9~$300                     │
│  n8n (Self-hosted) : 무료 (서버 비용만)              │
└─────────────────────────────────────────────────────┘
```

## 슬라이드 1-2: 왜 n8n인가?

### 도입 배경
```
문제점:
1. 보험 청구 심사 건당 30분~1시간 소요
2. 담당자마다 심사 기준 불일치
3. 사기 청구 탐지의 어려움
4. AI 도입 필요하나 단일 모델의 한계

해결책 → n8n + AI 앙상블
```

### n8n 선택 이유
```
✅ 다양한 AI API 직접 호출 가능 (OpenAI, Anthropic, Google)
✅ 데이터베이스(PostgreSQL) 직접 연동
✅ 조건 분기, 병렬 처리, 반복 지원
✅ 시각적 디버깅 및 실시간 모니터링
✅ 사내 보안 정책 준수 (데이터 외부 유출 없음)
```

## 슬라이드 1-3: n8n 기본 구성 요소

### 노드(Node) 기반 워크플로우
```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   Trigger    │ ─▶ │   Process    │ ─▶ │   Action     │
│   (시작점)    │    │   (처리)      │    │   (결과)      │
└──────────────┘    └──────────────┘    └──────────────┘
```

### 주요 노드 유형
| 노드 유형 | 예시 | 용도 |
|----------|------|------|
| **Trigger** | Webhook, Schedule | 워크플로우 시작 |
| **App** | Slack, PostgreSQL, HTTP Request | 외부 서비스 연동 |
| **Core** | IF, Switch, Merge, Code | 로직 제어 |
| **AI** | OpenAI, Anthropic | AI 분석 |

## 슬라이드 2-2: 왜 3개의 AI를 사용하는가?

### 단일 AI의 한계
```
예시 상황:
- 면책 기간(계약 후 90일) 내 청구 발생
- 정상적으로는 거절해야 함

단일 AI (GPT-4o만 사용):
→ GPT-4o가 면책 기간을 놓치고 "승인" 판정
→ 잘못된 지급 발생 가능
```

### 앙상블의 장점
```
같은 상황에서 3개 AI 판단:

  GPT-4o  : "승인" (신뢰도 85%)
  Claude  : "검토" (신뢰도 92%) ← 면책기간 감지!
  Gemini  : "검토" (신뢰도 88%) ← 면책기간 감지!

투표 결과:
→ "검토"가 다수결
→ 담당자 검토로 전달
→ 잘못된 자동 승인 방지!
```

### 앙상블 효과
```
┌────────────────────────────────────────┐
│ 단일 AI 정확도       : 약 85~90%       │
│ 3개 앙상블 정확도    : 약 95~98%       │
│ 정확도 향상          : +8~10%p         │
└────────────────────────────────────────┘
```

## 슬라이드 2-3: 앙상블 투표 시스템

### 가중 투표 방식
```javascript
// 각 AI의 추천에 가중치와 신뢰도를 곱하여 투표 점수 계산
votes = {
  APPROVE: (GPT가중치 × GPT신뢰도) + (Claude가중치 × Claude신뢰도) + ...
  REJECT:  ...
  REVIEW:  ...
}

// 가장 높은 점수가 최종 결정
finalDecision = max(votes)
```

### 합의 수준 (Consensus Level)
| 수준 | 조건 | 의미 | 처리 |
|------|------|------|------|
| **FULL** | 3개 모두 동일 | 만장일치 | 높은 신뢰, 자동 처리 |
| **STRONG** | 70% 이상 동의 | 강한 합의 | 자동 처리 가능 |
| **WEAK** | 그 외 | 의견 분분 | 인간 검토 권장 |

## 슬라이드 2-5: 병렬 처리의 효율성

```
순차 처리 (일반적 방식):

│                                                     │
│ 총 소요시간: 1,550ms                                 │
└─────────────────────────────────────────────────────┘

병렬 처리 (n8n 구현):
┌─────────────────────────────────────────────────────┐

│ Gemini  (450ms) ──┘                                 │
│                                                     │
│ 총 소요시간: 600ms (가장 느린 모델 기준)              │
│                                                     │
│ ⏱️ 약 60% 시간 단축!                                │
└─────────────────────────────────────────────────────┘
```

### [앙상블 필요성] - 2분 30초
```
왜 굳이 3개를 사용하느냐 질문하실 수 있습니다.
구체적인 예를 들어 설명드리겠습니다.

면책 기간 내에 접수된 청구가 있다고 가정합니다.
계약 후 90일 이내 발생한 질병은 보장하지 않는 조건입니다.
이 경우 당연히 거절 또는 검토로 분류해야 합니다.

그런데 GPT-4o가 이 조건을 놓치고 "승인"이라고 판단했습니다.
만약 GPT 하나만 사용했다면 잘못된 지급이 발생했을 것입니다.

하지만 앙상블 시스템에서는 다릅니다.
Claude와 Gemini는 면책 기간을 정확히 감지하고
둘 다 "검토"라고 판단했습니다.

투표 결과, "검토"가 다수가 되어
자동 승인 대신 담당자에게 전달됩니다.
단일 AI의 실수를 다른 AI들이 보완한 것입니다.

이런 상호 보완 효과로
단일 AI의 85~90% 정확도가
앙상블에서는 95~98% 수준으로 향상됩니다.
약 10%p의 정확도 향상입니다.

### [투표 시스템] - 2분
```
투표 방식을 조금 더 설명드리겠습니다.

단순 다수결이 아니라 '가중 투표' 방식입니다.
각 AI의 추천에 가중치와 신뢰도를 곱합니다.

예를 들어 GPT-4o가 "승인"을 85% 신뢰도로 추천하면
0.35 곱하기 0.85, 약 0.30점이 "승인" 항목에 더해집니다.

세 AI의 점수를 모두 합산하여
가장 높은 점수를 받은 항목이 최종 결정됩니다.

또한 '합의 수준'도 함께 확인합니다.
세 AI가 모두 같은 추천을 하면 "FULL", 만장일치입니다.
70% 이상이 동의하면 "STRONG"입니다.
그 외에는 "WEAK"로, 인간 검토를 더 강력히 권장합니다.

합의 수준이 낮을수록 담당자 개입이 필요하다는

# 부록: Q&A 예상 질문

## Q1. n8n이 장애가 발생하면 어떻게 되나요?
```
n8n은 모든 워크플로우 실행 로그를 저장합니다.
실패한 워크플로우는 자동 재시도 옵션이 있고,
중요 워크플로우는 Slack 알림을 설정하여 즉시 대응 가능합니다.
또한 n8n 자체를 이중화하여 고가용성을 확보할 수 있습니다.
```

## Q2. AI API 비용은 얼마나 드나요?
```
현재는 시뮬레이션 모드로 비용이 발생하지 않습니다.

실제 API 연동 시 비용 예상:
- GPT-4o: 건당 약 30~40원
- Claude: 건당 약 25~35원
- Gemini: 건당 약 15~25원
- 합계: 건당 약 70~100원

월 1,000건 기준 약 7~10만원 수준입니다.
자동화로 절감되는 인건비에 비해 매우 저렴합니다.
```

## Q3. 기존 시스템과의 연동은 어떻게 하나요?
```
n8n은 REST API를 통해 기존 백엔드와 직접 통신합니다.
HTTP Request 노드로 모든 API 호출이 가능합니다.
PostgreSQL 노드로 데이터베이스 직접 연결도 지원합니다.
기존 시스템 수정 없이 n8n만 추가하면 됩니다.
```

## Q4. 보안은 안전한가요?
```
n8n은 사내 서버에 설치되어 외부에 데이터가 나가지 않습니다.
AI API 호출 시에도 필요한 최소 정보만 전송합니다.
모든 통신은 HTTPS로 암호화됩니다.
또한 역할별 접근 권한 관리도 가능합니다.
```

## Q5. 운영 인력은 몇 명 필요한가요?
```
초기 구축 후에는 모니터링 위주로 1명이 겸업 가능합니다.
워크플로우 수정이 필요할 때만 개발 인력이 투입됩니다.
n8n의 시각적 인터페이스 덕분에
개발자가 아니어도 간단한 수정은 가능합니다.
```

## Q6. AI가 잘못된 판단을 하면 어떻게 되나요?
```
세 가지 안전장치가 있습니다.

첫째, 앙상블 투표로 단일 AI 오류를 보완합니다.
둘째, 합의 수준이 낮으면 자동으로 인간 검토로 전달됩니다.
셋째, 고위험 건은 무조건 담당자 검토를 거칩니다.

AI는 의사결정 '보조' 역할이고,
최종 승인은 담당자 또는 시스템 규칙에 따릅니다.
```
