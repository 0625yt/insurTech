> 필요하면 간단한 JavaScript 코드도 넣을 수 있습니다.
>
> 저희가 주목한 건 AI API 연동입니다.
> Claude, GPT-4 같은 AI를 워크플로우 안에서 직접 호출할 수 있습니다.

>
> **1단계 - 데이터 입력**
> 청구 데이터가 들어오면 DB에 저장합니다.
>
> **2단계 - 병렬 처리 (3개)**
> OCR 문서 분석, 병원 정보 검증, 고객 이력 조회를 **동시에** 처리합니다.

> **4단계 - 사기 탐지**
> 여러 요소를 점수화해서 사기 위험도를 계산합니다.
>
> **5단계 - 조건 분기**
> 위험도가 낮으면 자동 승인, 높으면 수동 심사로 분기합니다.
>
> **6단계 - 후처리 (병렬)**
> DB 업데이트, 알림 발송, 결재 요청을 동시에 처리합니다.

```
│   • 조건에 따른 분기 처리 (위험도 점수 기준)             │
│   • 알림 발송, DB 저장 등 후처리 작업                    │
│                                                          │
│   ⚠️ 사람 검토가 필요한 영역                             │
│   ─────────────────────────────────                     │
│   • 복잡하거나 애매한 약관 해석                          │
│   • AI 모델 간 결과가 크게 다른 경우                     │
│   • 고위험/고액 건 (기준 금액 초과 시)                   │
│   • 새로운 유형의 케이스                                 │
│                                                          │
│   ❌ 자동화하기 어려운 영역                              │
│   ─────────────────────────────────                     │
│   • 학습되지 않은 새로운 패턴의 사기                     │
│   • 법적 판단이 필요한 분쟁 건                           │
│   • 고객과의 직접 소통 (민원 대응 등)                    │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**[대본]**
> n8n과 AI로 어디까지 자동화할 수 있는지 정리했습니다.
>
> **완전 자동화 가능한 영역**은,
> 문서 OCR, 규칙 기반 계산, API 호출, 조건 분기, 후처리 작업입니다.
> 이런 건 사람 개입 없이 처리할 수 있습니다.
>
> **사람 검토가 필요한 영역**은,
> 애매한 약관 해석, AI 결과 불일치, 고위험 건 등입니다.
> 시스템이 "이건 사람이 봐야 한다"고 판단하면 수동 심사로 넘깁니다.
>
> **자동화하기 어려운 영역**도 있습니다.
> 새로운 패턴의 사기, 법적 분쟁 건, 고객 민원 대응은
> AI가 처리하기 어렵습니다.
>
> 정리하면, 모든 업무를 자동화할 수는 없지만,
> 정형화된 반복 업무는 충분히 자동화할 수 있습니다.

# 📌 예상 질문 & 답변

---

### Q1: "n8n은 무료인가요?"

**[답변]**
> 네, 오픈소스라서 셀프 호스팅하면 무료입니다.
> Docker로 로컬에서 실행했습니다.
> 클라우드 버전(n8n.cloud)은 유료입니다.

---

### Q2: "AI API 비용은 어느 정도인가요?"

**[답변]**
> Claude와 GPT-4 기준으로, 청구 건당 약 50~100원 정도 예상됩니다.
> (입력 토큰 + 출력 토큰 비용)
>
> 다만 이번 시연에서는 실제 API 호출이 아닌 시뮬레이션을 사용했습니다.
> 워크플로우 구조를 보여드리기 위한 목적이었습니다.

---

### Q3: "실제 운영 환경에 적용할 수 있나요?"

**[답변]**
> 현재 수준은 POC(개념 증명) 단계입니다.
>
> 실제 운영에 적용하려면 추가 작업이 필요합니다.
> - 실제 AI API 연동 및 비용 검토
> - 보안 검토 (고객 정보가 외부 API로 전송되므로)
> - 예외 케이스 처리 강화
> - 모니터링 및 오류 대응 체계
>
> 작은 범위의 파일럿으로 시작해서 점진적으로 확대하는 것을 권장합니다.

---

### Q4: "AI가 틀리면 어떻게 되나요?"

**[답변]**
> 그래서 앙상블 방식을 사용했습니다.
> Claude와 GPT-4 두 모델의 결과를 비교해서,
> 결과가 크게 다르면 사람이 검토하도록 설계했습니다.
>
> 또한 고위험 건이나 고액 건은 위험도와 관계없이
> 무조건 수동 심사로 넘기도록 할 수 있습니다.

---

### Q5: "코딩을 전혀 모르는 사람도 n8n을 쓸 수 있나요?"

**[답변]**
> 기본적인 워크플로우는 드래그 앤 드롭으로 만들 수 있습니다.
>
> 다만 복잡한 로직(데이터 변환, 조건 분기 등)은
> 간단한 JavaScript 지식이 있으면 더 유연하게 구현할 수 있습니다.
>
> 저희가 만든 워크플로우에도 JavaScript 코드 노드가 들어가 있습니다.
